# Frontend

Компактный операторский экран для работы по одному товару.

Frontend намеренно тонкий:

- HTML/CSS/JS только рисуют экран и локальное UI-state
- business payload приходит из backend `/api/workspace`
- те же backend-данные потом должны форматироваться и для TG

## Что сейчас есть на экране

- верхняя карусель кейсов и быстрые действия
- workbook drawer для intake/import
- слева фото и analysis
- по центру чат по текущему кейсу
- справа реальный `Output` из backend-payload

## Экосбор в UI

Экосборный блок сейчас умеет:

- читать `current_case.eco_fee` из общего workspace payload
- переключать годы `2025 / 2026 / 2027`
- сворачиваться и раскрываться
- показывать компактный summary по выбранному eco-match без дубля `ЭКО к СТП / СТП + ЭКО`
- показывать все названия из БД по совпадению
- показывать сноски `<N>`
- показывать top-10 строк БД по совпадению
- открывать centered modal со всеми строками БД по совпадению
- открывать модалку с конкретной строкой БД для проверки оператором
- показывать `USD / EUR` от ЦБ РФ в подсказках

## Важный принцип

Если в UI чего-то не хватает, сначала меняем backend payload и contracts, а не дописываем отдельную фронтовую бизнес-логику.
